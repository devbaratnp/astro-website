# Google Calendar + Meet Booking Integration Plan

## Overview
Replace manual slot system with real Google Calendar availability and auto-generated Google Meet links for video consultations.

## Prerequisites (User Must Do)
1. Go to https://console.cloud.google.com → Create project (e.g. "astro-shreehari")
2. Enable **Google Calendar API**
3. Create **Service Account** → download JSON key file
4. Share your Google Calendar with the service account email (Settings → Share with specific people → "Make changes to events")
5. Upload the JSON key to server at `backend/config/gcal-service-account.json` (outside web root ideally)

## Files to Create

### `backend/lib/GoogleCalendar.php`
- PHP class for Google Calendar REST API via Service Account JWT auth
- Self-contained: uses JWT library built into PHP (no Composer needed)
- Uses `openssl_sign()` for JWT creation
- Methods:
  - `getAvailableSlots(date, timezone)` → queries freebusy API
  - `createEvent(details)` → creates event with Google Meet, returns meeting URL
  - `getEvents(date)` → list events for a date
- Timezone: `Asia/Kathmandu` (UTC+5:45)
- Auth: Service Account → request access token → call Calendar API

### `backend/api/appointments.php` (Update)
- **GET `?date=`** → if gcal credentials exist, query freebusy for available slots; fall back to current logic
- **POST** → save to DB + create Google Calendar event with:
  - Title: `परामर्श: [client name] - [service type]`
  - Description: all form details
  - Attendees: client email (if provided) + astrologer
  - ConferenceData: Google Meet enabled
  - Return `meeting_url` with Meet link

### `astro-shree-hari-source/src/pages/Appointment.jsx` (Update)
- Load available slots from Google Calendar via backend API
- Show real-time availability per selected date
- Display Google Meet link after booking confirmation
- Loading state while checking calendar availability

### `backend/config/google-calendar.example.json`
- Template for the service account JSON key with instructions

## Google Calendar API Endpoints Used
- `POST https://oauth2.googleapis.com/token` — Get access token from JWT
- `POST https://www.googleapis.com/calendar/v3/calendars/{calendarId}/events` — Create event (with `conferenceData: { createRequest: { requestId: "...", conferenceSolutionKey: { type: "hangoutsMeet" } } }`)
- `POST https://www.googleapis.com/calendar/v3/freeBusy` — Check availability

## JWT Auth Flow (No Composer Needed)
1. Create JWT header + payload signed with RS256 using `openssl_sign()`
2. Exchange JWT for access token via `https://oauth2.googleapis.com/token`
3. Use access token for Calendar API calls
4. Token expires in 1 hour — cache it or regenerate per request

## Key Implementation Details
- Service account email format: `xxx@xxx.iam.gserviceaccount.com`
- Calendar ID: the astrologer's Gmail or Google Workspace email
- Meet links are auto-generated by Google when `conferenceData.version` is set
- Free/busy response returns busy intervals → invert to get available slots
- Slot duration: 30 min default, configurable
- Buffer time between slots: 15 min (configurable)

## Security
- Service account JSON key stored outside web root (`../gcal-service-account.json`)
- No client-side Google API keys exposed
- Calendar API calls proxied through PHP backend
- Rate limiting on appointment creation
